Chicago Crime Over Time: 2001 - 2026


Overview

Chicago was recently ranked near the bottom in a national safety comparison (161st out of 182 U.S. cities), so we wanted to look past the headline ranking and dig into what might be driving safety concerns. Using the City of Chicago dataset “Crimes – 2001 to Present”, our goal was to build a visualization that shows where crime is more concentrated across the city and which crime categories are most common over time. This breakdown can help locals and tourists make more informed choices about where they go and when, and it can also help policymakers identify which areas and crime types may need more targeted resources.


Because the full dataset contains over 8.5 million records, we used a smaller sample that’s realistic for the assignment. We randomly selected 2,000 incidents per year, creating a final dataset of about 52,000 records. We also simplified the data to only four fields that directly support our visualization goals: Crime, Year, Latitude, and Longitude, so the results are easier to interpret without unnecessary details. To improve reliability, we removed any rows missing valid values in those fields (invalid or missing coordinates, missing years, or blank crime types), leaving a cleaner subset that’s large enough to show patterns but small enough to run smoothly in an interactive visualization.


Rationale

We built our visualization around a Chicago map because we wanted to show not just which community areas have more crime, but where crime clusters within those areas. We considered an alternative design that used bar charts split by both crime type and community area, but that approach would have pushed everything into one dense view and encouraged overgeneralizing entire communities as “unsafe.” The map made it easier to see patterns of higher activity without flattening the story into one number per community.


For visual encodings, we mapped each incident using its latitude and longitude so the main information is communicated through position, which is one of the strongest ways to represent quantitative, spatial data. We then encoded crime type by color and added a legend so users could quickly connect colors to categories. This was chosen because it lets patterns show up naturally through visible clusters, instead of forcing users to compare many stacked or segmented bars across dozens of categories. We also decided to use a simple visual encoding of a circle to represent the data points, as it will allow for the complexity of the colors to be showcased.


Our implemented interactions were designed to reduce clutter and make the visualization easier to explore. First, we included legend-based filtering, where users can check off specific crimes to focus on a smaller subset. Since there are about 32 crime categories, filtering helps avoid confusion when many colors overlap. Second, we added a brush (select) tool so users can highlight a region on the map and immediately see a linked bar chart summarizing crimes for just that selected area. This gives users a way to zoom into detail without losing the overall citywide context. Third, we added a slider that allows users to choose a range between 2001 and 2026. Each option shows the crime rates within Chicago for that year. A slider easily allows users to see how crime trends have fluctuated throughout the years.


Overall, we chose these interactions because they support the overall goal while also giving the ability to deep-dive into certain trends. The bar chart-only alternative was rejected because it would have required heavy labeling and splitting across many communities and crime types, making it visually overwhelming. By combining a map with filtering and brushing, we kept the main view intuitive while still giving users a clear way to isolate and analyze specific crimes and locations when the full map becomes busy.
